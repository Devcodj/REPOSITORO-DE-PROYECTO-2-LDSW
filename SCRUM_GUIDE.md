# Guía Scrum (resumen personalizado)

**Roles**
- Product Owner (PO): define prioridades del backlog (Fernando).
- Scrum Master (SM): facilita y elimina impedimentos.
- Equipo de Desarrollo: 3-5 miembros full‑stack.

**Artefactos**
- Product Backlog: archivo `src/data/backlog.json` + `PRODUCT BACKLOG.docx`.
- Sprint Backlog: items seleccionados para el sprint (ver SPRINT_PLANNING.md).
- Incremento: versión funcional desplegable.

**Eventos**
- Sprint Planning (2 horas para sprint de 2 semanas).
- Daily Scrum (15 min diarios).
- Sprint Review (1 hora).
- Sprint Retrospective (45 min).

**Definition of Done (DoD)**
- Código funcional y probado.
- UI responsiva en móvil y escritorio.
- Documentación mínima (README, SPRINT_PLANNING.md).
- Tests unitarios básicos (cuando aplicable).
